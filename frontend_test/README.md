# フロントエンド問題

提供されたワイヤーフレームのデータと下記の実装要求を確認し、サイトの見た目や機能を実装してください。

要求する実装内容は、

- 基本実装
- 追加実装
- オリジナル実装

の３つに分けて記載されています。


## [ 基本実装 ]

ワイヤーフレームを参考に、HTML・CSSでサイトコーディングをおこないサイトを実装してください。

マルチデバイスでの表示を考慮したレスポンシブ対応をおこない、表示サイズが変わってもサイトのレイアウトが崩れないようにしてください。

イベント情報の表示に際して、下記３点

- イベントタイトル
- イベントのカテゴリー
- イベントの実施日
  
は、必ず表示するようにしてください。

イベントデータは、APIで一般提供されているため、APIからデータを取得しイベント情報の表示に活用してください。

*scheWD API ドキュメント*
https://click.ecc.ac.jp/ecc/webdesign/wdct/doc/schewd/

*scheWD API*
https://click.ecc.ac.jp/ecc/webdesign/wdct/api/v1/schewd/

## [ 追加実装 ]

追加で実装して欲しい機能を記載します。
追加機能はどれから実装しても構わいません。


### イベントの絞り込み機能
- イベントタイトルを入力することで、表示するイベントを絞り込んで表示するようにしてください。
- イベントのカテゴリー名を選択することで、イベントを絞り込んで表示するようにしてください。


### 並び順の変更機能
APIから提供されているイベント一覧のデータは、カテゴリーIDで昇順（1,2,3と数値が増えていく）に並んでいるため、フロント側の処理で、イベント実施日で新しい順（降順）や古い順（昇順 過去から）に表示の並び順を変更できるようにしてください。


### 詳細画面の表示
イベントを押した際に、イベントの詳細情報として説明文が表示されるようにすること。
*（ページの変更、画面内で表示のどちらでも構わない）*

## [ オリジナル実装 ]
上記基本実装と追加実装以外にも、自身で最適と思われる追加実装をおこなうことも可能とする。

例えば、
- 表示するイベントを年度別に絞り込むことができる
- イベントの表示をカレンダー表記に変更することができる
- 終了したイベントの背景をグレーアウトにして、イベントが終了したかどうかを見た目で分かるようにする

など
